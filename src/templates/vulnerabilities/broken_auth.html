Broken Authentication - Insecure Password Reset

This page intentionally demonstrates a broken authentication vulnerability. The application allows a password reset without verifying the user's identity.

Vulnerable Password Reset

An attacker only needs to know the victim's username to reset the password. There is no reset token, no identity verification, no OTP, and no ownership validation.

Why This Is Vulnerable

A secure password reset flow must prove that the person requesting the reset owns the account. This implementation does not perform any verification.

Possible Mitigations

  • Generate a unique password reset token.
  • Send the reset link only to the user's registered email address.
  • Set a short expiration time for the reset token.
  • Invalidate the token after one use.
  • Hash the new password before storing it.
